📊 SQL SELECT & Filtering Exercise – Summary

This project is a beginner-level SQL exercise focused on building foundational data querying skills using the employees table. The assignment introduced how to extract, filter, and organise data using core SQL clauses that are essential for any data analyst or database practitioner.

Purpose of the Assignment

The main goal of this exercise was to develop a strong understanding of how to retrieve meaningful information from a relational database using SQL. It focused on learning how to structure queries correctly and interpret expected outputs before executing them.

Key Skills Covered
Through this exercise, the following SQL concepts were practiced:

Using SELECT to retrieve all columns or specific fields from a table
Removing duplicate values using DISTINCT
Sorting results using ORDER BY in both ascending and descending order
Limiting results using LIMIT to control output size
Filtering data using WHERE conditions
Combining multiple conditions using AND, OR, NOT, and IN
Predicting query results before execution to strengthen analytical thinking

Dataset Used
All questions were based on a single dataset: the employees table, which contained information such as:

Employee ID
First and last names
Department
Salary
Hire date
City

Task Structure
The assignment consisted of 10 structured questions divided into two sections:

Selecting & Sorting Data, focusing on retrieving, ordering, and limiting results
Filtering with WHERE Clause, focusing on narrowing down results using conditions

For each question, the learner was required to:

Write the SQL query using correct syntax and formatting
Manually predict and draw the expected output table
Ensure correct column names and structured results
 
Overall Learning Outcome
By completing this exercise, I strengthened my ability to:

Read and interpret SQL queries
Understand how filtering and sorting affect data output
Structure clean and correct SQL statements
Think like a data analyst by predicting results before execution

This foundational exercise helped me to build essential skills required for more advanced data analysis, reporting, and database querying tasks.

📊 SQL Aggregate Functions & Operators

This project is a continuation of my SQL learning journey and focuses on applying aggregate functions and operators to analyse structured datasets. The exercise moved beyond basic querying and introduced more advanced techniques for summarising, grouping, and interpreting data across multiple tables.

Purpose of the Assignment

The main goal of this exercise was to build a deeper understanding of how SQL can be used to analyse data at scale. Instead of working with single rows, I was required to work with grouped data, perform calculations across datasets, and extract insights that support decision-making.

How the Exercise Was Structured

The exercise was divided into multiple sections, each using a different dataset representing real-world scenarios such as students, courses, enrollments, salaries, and projects. Each dataset required a different analytical approach, but all tasks followed a similar pattern of identifying relevant data, applying the correct SQL functions, and producing structured outputs.

I was instructed to perform tasks such as calculating averages, summing values, counting records, filtering grouped results, and identifying maximum and minimum values. In more complex questions, I had to combine multiple SQL concepts such as GROUP BY, HAVING, and aggregate functions in a single query.

How I Tackled the Exercise

To complete the assignment, I approached each question systematically by first understanding what the question was asking, then identifying the correct dataset, and finally applying the appropriate SQL logic. I made use of aggregate functions such as SUM, AVG, COUNT, MAX, and MIN to summarise data.

I also applied filtering techniques using WHERE for row-level filtering and HAVING for group-level filtering. Logical operators such as AND, OR, and BETWEEN were used to refine results further. In addition, I used column aliases with the AS keyword to ensure that outputs matched the expected format and were easy to interpret.

Key Learning Outcome

Through this exercise, I developed a strong understanding of how aggregate functions transform raw data into meaningful insights. I learned the difference between WHERE and HAVING, and how GROUP BY is used to structure grouped datasets before applying calculations. I also improved my ability to combine multiple SQL concepts in a single query, which is essential for real-world data analysis.

Most importantly, I learned how to think in terms of data patterns and summaries rather than individual records. This helped me start developing a more analytical mindset, which is critical for roles in data science, business intelligence, and database management.
